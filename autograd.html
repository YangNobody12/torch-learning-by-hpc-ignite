🎯 Autograd คืออะไร?

เมื่อ train neural networks สิ่งที่ใช้บ่อยที่สุดคือ back propagation ใน algorithm นี้ parameters (weights ของ model) จะถูกปรับตาม gradient ของ loss function

PyTorch มี built-in differentiation engine ชื่อ torch.autograd ที่ support การคำนวณ gradient โดยอัตโนมัติ

📊 Computational Graph

เมื่อเราสร้าง tensors ที่มี requires_grad=True PyTorch จะสร้าง computational graph ขึ้นมา

x (input)
requires_grad=True
w (weights)
requires_grad=True
b (bias)
requires_grad=True
↓
MatMul + Add
Forward Pass
↓
Loss Function
Cross Entropy
↓
loss
Backward Pass → Gradients
# สร้าง tensors พร้อม requires_grad x = torch.ones(5) # input y = torch.zeros(3) # expected output w = torch.randn(5, 3, requires_grad=True) b = torch.randn(3, requires_grad=True) # Forward pass z = torch.matmul(x, w) + b loss = torch.nn.functional.binary_cross_entropy_with_logits(z, y)

📐 การคำนวณ Gradients

เรียก loss.backward() เพื่อคำนวณ gradient แล้วดูค่าได้จาก .grad

loss.backward()
→
w.grad
→
b.grad
# คำนวณ gradient loss.backward() print(w.grad) # tensor([[0.1224, 0.0324, 0.2692], ...]) print(b.grad) # tensor([0.1224, 0.0324, 0.2692])

⚠️ ข้อควรรู้

  • เราสามารถได้ grad เฉพาะ leaf nodes ที่มี requires_grad=True
  • เราสามารถเรียก backward() ได้ครั้งเดียวเท่านั้น ถ้าต้องการเรียกอีกต้องใช้ retain_graph=True

🚫 การปิด Gradient Tracking

บางครั้งเราไม่ต้องการ track gradient เช่นตอนทำ inference (โมเดล train แล้ว)

วิธีที่ 1: torch.no_grad()

with torch.no_grad(): z = torch.matmul(x, w) + b print(z.requires_grad) # False

วิธีที่ 2: detach()

z = torch.matmul(x, w) + b z_det = z.detach() print(z_det.requires_grad) # False

💡 ทำไมต้องปิด Gradient Tracking?

  • Speed: คำนวณเร็วขึ้นเพราะไม่ต้องเก็บ computation graph
  • Freeze: ใช้ freeze parameters บางตัว

🔁 Gradient Accumulation

PyTorch accumulates gradients - ค่า gradient จะถูกบวกเพิ่มเรื่อยๆ ถ้าเรียก backward หลายครั้ง

# เรียก backward ครั้งแรก out.backward(torch.ones_like(out)) print(inp.grad) # tensor([[4., 2., 2., ...]]) # เรียก backward ครั้งที่สอง (บวกเพิ่ม) out.backward(torch.ones_like(out)) print(inp.grad) # tensor([[8., 4., 4., ...]]) # บวกเพิ่มแล้ว! # ต้อง zero_grad ก่อนเรียกอีกครั้ง inp.grad.zero_() out.backward(torch.ones_like(out)) print(inp.grad) # tensor([[4., 2., 2., ...]]) # เริ่มใหม่
